# Feature Specification: 语音播报

**Feature Branch**: `002-voice-announcements`  
**Created**: 2026-03-04  
**Status**: Draft  
**Input**: User description: "我希望能增加语音播报，具体来说：每个项目开始前可以播报自定义内容1（默认为项目名称），Rest开始时可以播报自定义内容2（默认为"休息"），Worker开始时可以播报自定义内容3（默认为"继续"）。这样一来，每个项目第一组Work开始的时候播报的是自定义内容1，后面每组Work开始的时候播报的是自定义内容3。播报的声音要自然准确。"

## Clarifications

### Session 2026-03-04

- Q: 语音播报与现有 Work/Rest 开始音效的时序关系？ → A: 语音播报替代 Work/Rest 开始音效（倒计时音效、Session 完成音效保留不变）
- Q: Session 完成时是否需要语音播报？ → A: 增加 Session 完成播报，支持自定义文本（默认"训练完成"）
- Q: 用户输入非中文播报文本时如何处理？ → A: 自动检测文本语言，选择匹配语音（混合内容以主要语言为准）

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 计时阶段自动语音播报 (Priority: P1)

用户在使用 Session Timer 进行训练时，应用在每个关键阶段切换时自动播报语音提示，让用户无需查看屏幕即可知晓当前状态。具体播报规则如下：

- **Block 首组 Work 开始时**：播报该 Block 的"开始播报内容"（默认为 Block 名称，如"深蹲"）
- **Rest 开始时**：播报"休息播报内容"（默认为"休息"）
- **后续组 Work 开始时**（非首组）：播报"继续播报内容"（默认为"继续"）
- **Session 完成时**：播报"完成播报内容"（默认为"训练完成"）

这样用户在健身、练琴等双手忙碌的场景下，通过语音即可清楚知道当前应该做什么，也能明确知道整个训练何时结束。

**Why this priority**: 语音播报是本功能的核心价值，没有它其他配置功能毫无意义。即使使用默认播报内容（Block 名称 / "休息" / "继续"），功能也立即可用。

**Independent Test**: 创建一个含有多个 Block 的 Session，启动计时，验证每个阶段切换时能听到正确的语音播报。

**Acceptance Scenarios**:

1. **Given** 用户启动了一个包含 Block "深蹲"（3组）的 Session，**When** 第一组 Work 阶段开始，**Then** 播报语音"深蹲"
2. **Given** 第一组 Work 倒计时结束，**When** 进入 Rest 阶段，**Then** 播报语音"休息"
3. **Given** 第一组 Rest 倒计时结束，**When** 进入第二组 Work 阶段，**Then** 播报语音"继续"
4. **Given** 第三组 Rest 结束后进入下一个 Block "卧推"，**When** 新 Block 的第一组 Work 开始，**Then** 播报语音"卧推"
5. **Given** 所有 Block 的所有组均已完成，**When** Session 完成，**Then** 播报语音"训练完成"
6. **Given** 手机音量已调至可听水平且音效开关为开启状态，**When** 阶段切换发生，**Then** 语音播报清晰可辨、发音自然

---

### User Story 2 - 自定义播报内容 (Priority: P2)

用户可以为四种播报场景分别设置自定义文本，替代默认内容。例如，用户可能希望 Block 开始时播报"准备开始深蹲"而非简单的"深蹲"，或者休息时播报"放松一下"而非"休息"。

**Why this priority**: 自定义内容提供了更好的个性化体验，但即使没有此功能，默认播报内容已足够使用。

**Independent Test**: 在 Block 编辑界面修改三种播报内容的自定义文本，启动计时后验证播报的是自定义内容而非默认值。

**Acceptance Scenarios**:

1. **Given** 用户在编辑 Block 界面，**When** 展开语音播报设置区域，**Then** 可以看到三个文本输入项，分别对应"开始播报"、"休息播报"、"继续播报"，且各自有默认提示文本；Session 完成播报在 Session 级别设置
2. **Given** 用户将"开始播报"设置为"准备开始深蹲"，**When** 该 Block 的首组 Work 开始，**Then** 播报"准备开始深蹲"
3. **Given** 用户清空了"休息播报"的自定义文本，**When** Rest 阶段开始，**Then** 播报默认值"休息"
4. **Given** 用户修改了播报内容并保存，**When** 下次打开该 Block 编辑界面，**Then** 之前设置的自定义内容已被保留

---

### User Story 3 - 语音播报开关 (Priority: P3)

用户可以全局开启或关闭语音播报功能。关闭后，所有语音播报停止，但原有的音效提示不受影响。

**Why this priority**: 开关控制是基本的用户体验保障，让不需要语音播报的用户可以关闭它，但优先级低于核心播报功能本身。

**Independent Test**: 在设置中关闭语音播报开关，启动计时验证无语音播报；重新开启后验证语音播报恢复。

**Acceptance Scenarios**:

1. **Given** 语音播报开关为关闭状态，**When** 阶段切换发生，**Then** 不播报任何语音，但原有音效照常播放
2. **Given** 语音播报开关为开启状态（默认），**When** 阶段切换发生，**Then** 按规则正常播报语音
3. **Given** 用户在计时过程中关闭语音播报，**When** 下一次阶段切换发生，**Then** 不再播报语音

---

### Edge Cases

- 播报内容为空字符串时：使用对应的默认值（Block 名称 / "休息" / "继续"）
- 播报文本过长时（超过 50 个字符）：仍然完整播报，但在编辑界面提示用户建议简短
- 阶段切换过快时（如 Work 或 Rest 时长很短）：当前播报被中断，立即开始新阶段的播报
- 手机处于静音模式时：语音播报行为与现有音效行为保持一致（跟随系统音效设置）
- 手机处于后台时：语音播报应在后台正常工作（与现有音效行为一致）
- 用户正在播放音乐时：语音播报应与音乐混音播放，播报期间可适当压低音乐音量
- Block 名称包含特殊字符或数字时：语音引擎应能正确处理并朗读
- 播报文本为纯英文（如"Squat"）时：自动使用英文语音朗读
- 播报文本为中英混合（如"C大调 Scale"）时：以主要语言的语音朗读

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统必须在 Block 首组 Work 开始时播报"开始播报内容"（默认为 Block 名称）
- **FR-002**: 系统必须在 Rest 阶段开始时播报"休息播报内容"（默认为"休息"）
- **FR-003**: 系统必须在非首组 Work 开始时播报"继续播报内容"（默认为"继续"）
- **FR-004**: 系统必须在 Session 完成时播报"完成播报内容"（默认为"训练完成"）
- **FR-005**: 用户必须能够为每个 Block 分别设置三种播报场景（开始、休息、继续）的自定义文本，并能为 Session 设置完成播报的自定义文本
- **FR-006**: 自定义文本为空时，系统必须使用对应的默认值进行播报
- **FR-007**: 系统必须提供全局语音播报开关，默认为开启
- **FR-008**: 语音播报必须使用自然、清晰的语音合成，发音准确
- **FR-009**: 语音播报必须自动检测文本语言并选择匹配的语音进行朗读；混合语言内容以主要语言为准
- **FR-010**: 语音播报必须替代现有的 Work/Rest 开始音效；倒计时音效和 Session 完成音效保持不变
- **FR-011**: 语音播报的后台行为必须与现有音效保持一致
- **FR-012**: 当阶段快速切换时，系统必须中断当前播报并开始新播报
- **FR-013**: 自定义播报设置必须随 Block/Session 数据持久化存储，支持 iCloud 同步

### Key Entities

- **Block（扩展）**: 在现有 Block 模型基础上增加三个可选的播报文本属性——开始播报文本、休息播报文本、继续播报文本。当属性为空时使用默认值。
- **Session（扩展）**: 在现有 Session 模型基础上增加一个可选的完成播报文本属性。当属性为空时使用默认值"训练完成"。
- **语音播报开关**: 全局设置项，控制语音播报功能的启用/禁用状态。

## Assumptions

- 语音播报使用设备内置的语音合成能力，无需网络连接
- 语音播报自动检测文本语言并匹配语音；中英混合内容以主要语言为准
- 播报音量跟随系统媒体音量
- 语音播报开关状态在应用重启后保持
- 语音播报启用时替代 Work/Rest 开始音效；语音播报关闭时恢复原有音效

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 用户在无需查看屏幕的情况下，仅通过语音播报即可准确判断当前训练阶段（Block 名称、休息、继续），准确率达到 100%
- **SC-002**: 语音播报在阶段切换后 0.5 秒内开始播放，用户感知无明显延迟
- **SC-003**: 语音播报在前台和后台场景下均能正常工作
- **SC-004**: 自定义播报内容从设置到生效的操作步骤不超过 3 步（打开编辑 → 输入文本 → 保存）
- **SC-005**: 语音播报启用时替代 Work/Rest 开始音效，关闭时原有音效自动恢复；倒计时和完成音效始终正常工作
